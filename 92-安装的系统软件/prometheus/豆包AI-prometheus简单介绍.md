# Prometheus 简单介绍
Prometheus 是一个开源的监控系统，主要用来收集、存储和查询时序数据（时间序列数据），特别适合监控服务器、容器和微服务。

核心特点：
- 数据模型：以时间序列（metric）为基础，每个指标都有名称和标签，方便灵活查询
- 采集方式：通过 HTTP 主动拉取（pull）目标暴露的指标数据
- 存储：本地时序数据库，适合高写入低查询延迟的场景
- 查询语言：PromQL，强大灵活，支持聚合、过滤、数学运算等
- 告警：内置告警规则引擎，通过 Alertmanager 处理和发送告警
- 可视化：通常与 Grafana 配合使用，构建漂亮的监控仪表盘
- 自动发现：支持 Kubernetes、Consul 等环境的服务自动发现

工作流程：
1. Prometheus 定期从目标（如应用、服务器）拉取指标
2. 数据存储在本地时序数据库
3. 用户通过 PromQL 查询数据，在 Grafana 上展示
4. 触发告警规则时，Alertmanager 负责去重、分组和发送通知

适合场景：
- 云原生和微服务环境
- Kubernetes 集群监控
- 需要自定义监控指标的应用
- 希望有高度灵活性和可扩展性的监控系统

如果你在使用 Kubernetes 或需要自定义监控指标，Prometheus 通常是首选方案。
