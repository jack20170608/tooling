# Linux tree 命令详解

### 一、tree 命令核心作用

`tree` 是 Linux 系统中一款直观的 **目录结构可视化工具**，核心功能是将指定目录下的文件和子目录以 **树状图形式** 递归列出，清晰展示目录层级关系，无需手动逐层查看目录内容，极大提升查看目录结构的效率。常用于文档整理、项目结构梳理、教学演示等场景。

默认情况下，部分 Linux 系统（如 CentOS、Ubuntu 最小安装版）未预装 tree 命令，可通过包管理器快速安装：



* Ubuntu/Debian 系列：`sudo apt-get install tree`

* CentOS/RHEL 系列：`sudo yum install tree`

* Fedora 系列：`sudo dnf install tree`

### 二、基本语法



```
tree \[选项] \[目录路径]
```



* **选项**：控制树状图的显示内容、格式（如是否显示隐藏文件、文件大小等）；

* **目录路径**：指定要查看结构的目标目录，默认不指定时，显示当前工作目录的结构；

* 若仅输入 `tree`，则直接以树状图展示当前目录下所有文件和子目录（包含一级子目录下的内容，递归到底）。

### 三、常用选项与场景示例

tree 命令的选项丰富，可根据需求灵活组合，以下是高频使用的选项及典型场景：

#### 1. 基础显示：默认与指定目录



* **默认显示当前目录**：直接输入 `tree`，会从当前目录开始，展示所有层级的文件和子目录，目录以 `├──` 或 `└──` 标记，文件直接列出名称。



```
tree
```

示例输出（简化）：



```
.

├── doc

│   ├── report.txt

│   └── manual.pdf

├── photo

│   ├── 202401.jpg

│   └── 202402.jpg

└── readme.md

2 directories, 4 files
```



* **指定目录显示**：在命令后加目录路径，查看目标目录的结构（无需切换到该目录）。

  示例：查看 `/home/user/project` 目录结构



```
tree /home/user/project
```

#### 2. 控制显示层级：限制递归深度

当目录层级较深（如项目代码目录），默认显示所有层级会导致输出过长，可通过 `-L`（大写 L，Level）选项指定显示的 **最大层级**。



* 语法：`tree -L [层级数] [目录路径]`

* 示例 1：显示当前目录下 **1 级** 内容（仅当前目录的直接子目录和文件，不深入子目录内部）



```
tree -L 1
```

示例输出：



```
.

├── doc

├── photo

└── readme.md

2 directories, 1 file
```



* 示例 2：显示 `/var/log` 目录下 **2 级** 内容（查看日志目录的浅层结构，避免输出冗余）



```
tree -L 2 /var/log
```

#### 3. 显示隐藏文件（以 “.” 开头的文件 / 目录）

Linux 中隐藏文件 / 目录默认以 “.” 开头，tree 命令默认不显示，需加 `-a`（all）选项。



* 示例：显示当前目录下所有内容（包含隐藏文件，如 `.bashrc`、`.git` 目录）



```
tree -a
```

示例输出（含隐藏内容）：



```
.

├── .bashrc

├── .git

│   ├── config

│   └── HEAD

├── doc

└── readme.md

2 directories, 4 files
```

#### 4. 显示文件大小与详细信息

默认仅显示文件名 / 目录名，若需查看文件大小、权限、修改时间等详细信息，可结合以下选项：



* `-s`（size）：显示每个文件的 **大小**（单位：字节，可配合 `-h` 优化显示）；

* `-h`（human-readable）：将文件大小转换为人类易读格式（如 KB、MB、GB）；

* `-p`（permissions）：显示文件 / 目录的 **权限**（如 `-rwxr--r--`）；

* `-D`（date）：显示文件 / 目录的 **最后修改时间**。

* 示例 1：显示当前目录内容，并标注文件大小（易读格式）



```
tree -sh
```

示例输出：



```
.

├── \[4.0K]  doc

│   ├── \[ 512]  report.txt

│   └── \[2.3M]  manual.pdf

├── \[4.0K]  photo

│   ├── \[1.2M]  202401.jpg

│   └── \[980K]  202402.jpg

└── \[ 384]  readme.md

2 directories, 4 files
```



* 示例 2：显示 `/etc` 目录 1 级内容的权限、修改时间和大小



```
tree -L 1 -pDsh /etc
```

#### 5. 只显示目录（排除文件）

若仅需梳理目录层级，无需显示文件，可加 `-d`（directory-only）选项，仅列出所有目录。



* 示例：显示当前目录下的所有目录结构（不包含任何文件）



```
tree -d
```

示例输出：



```
.

├── doc

└── photo

2 directories, 0 files
```

#### 6. 输出到文件（保存目录结构）

若需将目录结构保存为文档（如项目结构说明），可通过重定向符号 `>` 将输出写入文件，而非直接显示在终端。



* 示例：将当前目录的 2 级结构（含文件大小）保存到 `dir_structure.txt` 文件中



```
tree -L 2 -sh > dir\_structure.txt
```



* 后续可通过 `cat dir_structure.txt` 或文本编辑器查看保存的内容。

#### 7. 其他实用选项



* `-F`（classify）：在目录后加 `/`，可执行文件后加 `*`，符号链接后加 `@`，便于快速区分文件类型；

  示例：`tree -F`，输出中 `doc/`（目录）、`script.sh``*`（可执行文件）、`link.txt@`（符号链接）；

* `-P`（pattern）：按文件名匹配显示（仅显示符合通配符模式的文件 / 目录）；

  示例：显示当前目录下所有以 `.txt` 结尾的文件及所在目录结构：`tree -P "*.txt"`；

* `-I`（ignore）：按文件名匹配排除（不显示符合通配符模式的文件 / 目录）；

  示例：显示当前目录结构，但排除所有 `.log` 文件：`tree -I "*.log"`。

### 四、注意事项



1. **目录权限**：若目标目录存在权限限制（如 `/root` 目录），普通用户执行 tree 会提示 “Permission denied”，需加 `sudo` 提升权限（如 `sudo tree /root`）；

2. **大规模目录**：对包含上万文件的目录（如系统根目录 `/`）执行 tree，会产生海量输出，建议结合 `-L` 限制层级（如 `tree -L 2 /`），避免终端卡顿；

3. **符号链接循环**：若目录中存在符号链接指向父目录（如 `ln -s .. ./link`），tree 会自动检测并标注 “\[recursive, not followed]”，避免无限递归；

4. **输出编码**：部分终端若编码不匹配（如中文目录显示乱码），可先执行 `export LANG=en_US.UTF-8` 或 `export LANG=zh_CN.UTF-8` 调整编码。
