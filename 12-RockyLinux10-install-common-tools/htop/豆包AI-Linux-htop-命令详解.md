# Linux htop 命令详解

### 一、htop 命令核心作用

`htop` 是 Linux 系统中一款 **交互式进程监控工具**，是传统 `top` 命令的增强版。它以直观的彩色界面实时展示系统进程状态，支持鼠标操作、进程搜索、排序、kill 进程等功能，相比 `top` 更易用、信息更丰富，核心作用包括：



1. **实时监控系统资源**：动态显示 CPU 使用率、内存占用、交换分区（Swap）使用情况、系统负载等核心指标；

2. **管理进程**：查看所有进程的 PID、用户、CPU / 内存占用率、运行状态等信息，支持快速终止（kill）、调整优先级（nice）进程；

3. **自定义显示**：可根据需求筛选进程、调整界面布局、隐藏无关信息，适配不同监控场景（如服务器运维、程序调试）。

### 二、安装方法（默认未预装）

大部分 Linux 发行版需手动安装 htop，通过系统包管理器即可快速完成：



| 系统系列           | 安装命令                                                                 |
| -------------- | -------------------------------------------------------------------- |
| Ubuntu/Debian  | `sudo apt update && sudo apt install htop`                           |
| CentOS/RHEL 7  | `sudo yum install epel-release && sudo yum install htop`（需先装 EPEL 源） |
| CentOS/RHEL 8+ | `sudo dnf install htop`                                              |
| Fedora         | `sudo dnf install htop`                                              |
| Arch Linux     | `sudo pacman -S htop`                                                |

安装完成后，直接在终端输入 `htop` 即可启动工具。

### 三、界面组成解析

启动 htop 后，界面分为 **顶部系统信息区** 和 **底部进程列表区**，各区域信息清晰直观：

#### 1. 顶部系统信息区（3 行核心指标）



| 行号    | 显示内容                                          | 说明示例                                                                                          |
| ----- | --------------------------------------------- | --------------------------------------------------------------------------------------------- |
| 第 1 行 | 系统名称、内核版本、运行时间、当前用户数、负载平均值（1/5/15 分钟）         | `Ubuntu 22.04.3 LTS - 5.15.0-88-generic - 2d 3h 10m - 2 users - load average: 0.85 0.62 0.58` |
| 第 2 行 | CPU 使用率统计（按核心拆分，不同颜色代表不同类型任务）                 | \`CPU\[                                                                                       |
| 第 3 行 | 内存（Mem）和交换分区（Swap）使用情况（已用 / 总量，百分比，缓存 / 缓冲信息） | \`Mem\[                                                                                       |

#### 2. 底部进程列表区（列含义说明）

进程列表按列展示关键信息，默认列含义如下（可通过设置自定义列）：



| 列名      | 含义                                                |
| ------- | ------------------------------------------------- |
| PID     | 进程唯一标识（Process ID）                                |
| USER    | 运行该进程的用户                                          |
| PRI     | 进程优先级（Priority，数值越小优先级越高，内核动态调整）                  |
| NI      | 进程 nice 值（静态优先级调整值，范围 -20\~19，值越小优先级越高）           |
| VIRT    | 进程占用的虚拟内存总量（包括代码、数据、共享库、交换空间等）                    |
| RES     | 进程占用的物理内存（ Resident Memory，实际使用的物理内存，不包含交换）       |
| SHR     | 进程与其他进程共享的内存大小                                    |
| S       | 进程运行状态（R = 运行中，S = 睡眠，D = 不可中断睡眠，Z = 僵尸进程，T = 暂停） |
| %CPU    | 进程占用的 CPU 百分比（实时更新）                               |
| %MEM    | 进程占用的物理内存百分比                                      |
| TIME+   | 进程从启动到当前累计占用的 CPU 时间（精确到毫秒）                       |
| COMMAND | 启动进程的命令（如 `bash`、`python3`、`nginx`）               |

### 四、基本操作与快捷键

htop 支持 **鼠标操作**（点击表头排序、选中进程等）和 **键盘快捷键**，以下是高频操作（启动后按 `F1` 可查看完整快捷键列表）：

#### 1. 进程筛选与搜索



* **按用户筛选**：按 `U`（大写 U），在弹出的列表中选择目标用户，仅显示该用户的进程；

* **按关键词搜索**：按 `/`，输入关键词（如进程名、PID），匹配的进程会高亮显示，按 `ESC` 退出搜索；

* **按 PID 定位**：按 `F3`（或 `/` 后输入 PID），快速找到指定 PID 的进程。

#### 2. 进程排序



* 点击列表表头（如 `%CPU`、`%MEM`），可按该列升序 / 降序排序（再次点击切换顺序）；

* 快捷键：按 `P` 按 CPU 使用率排序，按 `M` 按内存使用率排序，按 `T` 按累计 CPU 时间排序。

#### 3. 进程管理（需 root 权限，普通用户仅能管理自己的进程）



* **终止进程**：选中目标进程（方向键移动），按 `F9`（或 `k`），在弹出的信号列表中选择 `SIGTERM`（默认，优雅终止）或 `SIGKILL`（强制终止，对应 `kill -9`），按 `Enter` 执行；

* **调整优先级**：选中进程，按 `F7`（降低 nice 值，提高优先级）或 `F8`（提高 nice 值，降低优先级），每次调整 1，范围 -20\~19（普通用户仅能调高 nice 值，即降低优先级）；

* **查看进程详情**：选中进程，按 `F2` 进入设置界面，或按 `s` 查看进程的系统调用（需安装 `strace` 工具）。

#### 4. 界面控制与设置



* **刷新频率调整**：按 `F2` 进入设置界面，选择 `Update frequency`，调整刷新间隔（默认 1 秒，范围 0.1\~10 秒）；

* **自定义列显示**：按 `F2` → `Display options` → `Columns`，可勾选 / 取消列（如隐藏 `SHR`、添加 `PPID` 父进程 ID）；

* **切换显示模式**：按 `F5` 切换为 “树状视图”，展示进程间的父子关系（便于排查进程依赖）；按 `F6` 选择排序字段（与点击表头功能一致）；

* **退出 htop**：按 `F10` 或 `q`。

### 五、实用场景示例

#### 1. 快速定位高 CPU 占用进程

当系统 CPU 使用率过高时，启动 htop 后：



1. 按 `P` 按 CPU 使用率降序排序，顶部进程即为高占用进程；

2. 选中该进程，按 `F9` 选择 `SIGTERM` 尝试优雅终止（若无法终止，选择 `SIGKILL` 强制终止）；

3. 若需分析进程用途，可按 `s` 查看其系统调用（需 `strace` 支持），或按 `l`（小写 L）查看进程打开的文件列表。

#### 2. 监控内存占用，排查内存泄漏



1. 按 `M` 按内存使用率排序，关注 `RES`（物理内存）列数值异常大的进程；

2. 观察 `%MEM` 列，若某进程内存占用持续增长（无下降趋势），可能存在内存泄漏；

3. 记录该进程 PID（如 1234），后续可通过 `ps aux | grep 1234` 或 `pstack 1234` 进一步分析。

#### 3. 按用户筛选进程（如查看 nginx 服务进程）



1. 按 `U`，在弹出的用户列表中选择 `nginx`（或运行 nginx 的用户）；

2. 界面仅显示 `nginx` 用户的进程，可快速查看该服务的所有进程状态（如是否有异常进程、CPU / 内存占用是否正常）。

### 六、注意事项



1. **权限问题**：普通用户仅能查看所有进程的基本信息，但无法终止 / 调整其他用户的进程；需管理系统级进程时，需用 `sudo htop` 启动（获取 root 权限）；

2. **资源消耗**：htop 本身对系统资源消耗极低（CPU 使用率通常 <1%，内存占用 <10MB），可长期后台运行监控；

3. **与 top 的区别**：相比 `top`，htop 优势在于支持鼠标操作、彩色界面、树状视图、更直观的资源展示，适合新手和需要高效管理的场景；若需在无图形环境的极简系统中使用，`top` 更通用（默认预装）；

4. **远程服务器使用**：通过 SSH 远程连接服务器后，启动 htop 即可实时监控，无需额外配置。
