# Linux tar 命令详解

### 一、tar 命令核心作用

`tar`（tape archive，磁带归档）是 Linux 系统中最常用的 **归档与压缩工具**，核心功能包括：



1. **归档**：将多个文件 / 目录打包成单个归档文件（通常以 `.tar` 为后缀，俗称 “tar 包”），不改变文件大小；

2. **压缩**：结合压缩算法（如 gzip、bzip2）将归档文件进一步压缩，减少存储空间（对应 `.tar.gz`、`.tar.bz2` 等格式）；

3. **解包 / 解压**：反向操作，将归档文件或压缩归档文件恢复为原始文件 / 目录。

### 二、基本语法



```
tar \[选项] \[归档文件名] \[待处理的文件/目录]
```



* **选项**：控制 tar 命令的行为（如打包、压缩、解包），是核心参数；

* **归档文件名**：指定生成或待处理的 tar 包名称（通常需明确后缀，方便识别格式）；

* **待处理的文件 / 目录**：打包时指需要加入归档的目标，解包时可省略（默认提取所有内容）。

### 三、常用选项与组合场景

tar 命令选项众多，实际使用中常以 **组合选项** 满足需求，以下是高频场景：

#### 1. 仅归档（不压缩）：生成 .tar 包



* **选项**：`-c`（create，创建归档） + `-f`（file，指定归档文件名，必须放在选项最后）

* **示例**：将 `doc/` 目录和 `report.txt` 文件归档为 `backup.tar`



```
tar -cf backup.tar doc/ report.txt
```



* **说明**：`-cf` 是最基础的打包组合，生成的 `.tar` 包仅打包不压缩，体积与原文件总和相近。

#### 2. 归档并压缩（常用格式）

tar 本身不具备压缩能力，需依赖外部压缩工具，通过选项调用对应算法：



| 压缩格式  | 选项组合         | 归档文件名后缀            | 特点              |
| ----- | ------------ | ------------------ | --------------- |
| gzip  | `-czf`       | `.tar.gz` 或 `.tgz` | 压缩速度快，应用最广泛     |
| bzip2 | `-cjf`       | `.tar.bz2`         | 压缩率比 gzip 高，速度慢 |
| xz    | `-cJf`（大写 J） | `.tar.xz`          | 压缩率最高，速度最慢      |



* **示例 1**：用 gzip 压缩 `photo/` 目录，生成 `photo.tar.gz`



```
tar -czf photo.tar.gz photo/
```



* **示例 2**：用 xz 压缩 `data.csv` 文件，生成 `data.csv.tar.xz`



```
tar -cJf data.csv.tar.xz data.csv
```

#### 3. 解包 / 解压（核心操作）



* **通用选项**：`-x`（extract，提取内容） + `-f`（指定待处理的 tar 包），根据压缩格式可选加 `-z`/`-j`/`-J`（也可省略，tar 会自动识别格式）；

* **常用组合**：`-xf`（万能解包，自动适配压缩格式），推荐优先使用；

* **指定解压目录**：加 `-C`（大写 C，指定目标目录，需提前创建）。

* **示例 1**：解压 `backup.tar.gz` 到当前目录



```
tar -xf backup.tar.gz  # 无需手动加 -z，tar 自动识别 gzip 格式
```



* **示例 2**：将 `docs.tar.bz2` 解压到 `/home/user/docs/` 目录



```
\# 先确保目标目录存在，不存在则创建：mkdir -p /home/user/docs/

tar -xf docs.tar.bz2 -C /home/user/docs/
```

#### 4. 查看归档文件内容（不提取）



* **选项**：`-t`（list，列出归档内的文件 / 目录） + `-f`

* **示例**：查看 `software.tar.xz` 中有哪些文件



```
tar -tf software.tar.xz
```



* **进阶**：加 `-v`（verbose，详细信息）可显示文件权限、大小、修改时间等



```
tar -tvf software.tar.xz
```

#### 5. 其他实用选项



* `-v`（verbose）：显示命令执行过程（如打包 / 解压时列出处理的文件，便于排查问题）；

* `-p`（preserve permissions）：保留文件原始权限（解压时常用，避免权限丢失）；

* `-exclude`：打包时排除指定文件 / 目录（如排除 `tmp/` 目录下的 `.log` 文件）



```
tar -czf project.tar.gz --exclude=tmp/\*.log project/
```

### 四、注意事项



1. **选项顺序**：`-f` 必须紧跟归档文件名，且通常放在选项最后（如 `tar -czf xxx.tar.gz` 正确，`tar -cfz xxx.tar.gz` 错误）；

2. **目录路径**：打包目录时，若路径为 `./dir/`（相对路径），解压后会保留层级；若为 `/absolute/dir/`（绝对路径），解压后会按绝对路径恢复（需注意权限）；

3. **大文件处理**：对于超过 4GB 的文件，需确保 tar 版本支持（现代 Linux 系统的 tar 均支持大文件，无需额外配置）；

4. **覆盖问题**：解压时若目标目录已有同名文件，默认直接覆盖，无提示（若需安全验证，可加 `-k` 选项禁止覆盖）。
